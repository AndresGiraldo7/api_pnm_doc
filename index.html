PNM API

API de consulta de numeración telefónica. Permite identificar portabilidad, operador, región y tipo de línea a partir de un número telefónico.

Autenticación

Todos los endpoints requieren autenticación HTTP Basic. Las credenciales se envían en el header Authorization como base64 de usuario:contraseña.

Header requerido 
Authorization: Basic <base64(usuario:contraseña)>
ⓘ Las peticiones sin credenciales o con credenciales incorrectas retornan HTTP 401.

Endpoints

POST /pnm

Consulta individual o batch de números telefónicos. Retorna información de operador, portabilidad, región, tipo de línea y blacklist.

Body (application/json) 
{
  "phones": [
    {
      "phone": "5589876576",
      "account": "1"
    }
  ]
}

POST /BulkPnm

Endpoint optimizado para volúmenes grandes de números. Presumiblemente procesa lotes más grandes de forma asíncrona o con mayor tolerancia. El tamaño máximo de lote y comportamiento bajo carga paralela están por determinar.

⚠ Este endpoint no es funcional actualmente. Tamaño máximo de lote por determinar. Validar comportamiento antes de producción.
Body (application/json) 
{
  "phones": [
    { "phone": "5589876576", "account": "1" },
    { "phone": "3312345678", "account": "1" },
    { "phone": "8112345678", "account": "1" }
  ]
}
cURL 
curl -X POST https://pp1.bonsaif.com/BulkPnm \
  -u usuario:contraseña \
  -H "Content-Type: application/json" \
  -d '{"phones": [{"phone":"5589876576","account":"1"},{"phone":"3312345678","account":"1"}]}'
GET /phonePnm

Alternativa GET para consultar un solo número via query param. Más simple de invocar desde browser o herramientas sin soporte de body. Solo acepta un número por request.

Query params

ParamTipoDescripción
phone*stringNúmero telefónico de 10 dígitos
URL de ejemplo 
GET https://pp1.bonsaif.com/phonePnm?phone=5555555555
⚠ Este endpoint retorna una estructura diferente a /pnm — los datos están anidados en result.planNacional, result.portabilidad y result.blackList. El campo code retorna como string, no como integer.

Schema: ApiResponse

Estructura de respuesta común a todos los endpoints.

CampoTipoDescripción
codeinteger Código de respuesta HTTP interno. Ej: 200
timefloat Tiempo de procesamiento en segundos. Ej: 0.008
tag string Identificador del módulo que respondió. Ej: "pnm.js"
dataPnmRecord[]Array de resultados por número consultado

Schema: PnmRecord

Objeto retornado por número dentro del array data.

CampoTipoDescripción
phone string Número telefónico consultado
account string Cuenta del solicitante
lada integer Código de área (LADA). Ej: 55
serie integer Serie del número
serieInicial integer Inicio del rango de serie
serieFinal integer Fin del rango de serie
modalidad enum Tipo de servicio
FIJO | MOVIL
razonsocial string Nombre legal del operador telefónico
poblacion string Localidad del número
municipio string Municipio del número
estado string Estado del número. Ej: "CDMX"
region integer Región geográfica
tipored enum Tipo de red
FIJO | MOVIL
id_zh integer Identificador de zona horaria
carrier string Identificador interno del carrier
id_estado integer Identificador numérico del estado
id_compania_tel integer Identificador numérico de la compañía
numeracion string Últimos dígitos del número
portability enum Indicador de portabilidad
0 = no portado · 1 = portado
ds_portability string Descripción del estado de portabilidad
blacklistenumIndicador de lista negra
0 = no está · 1 = está
